## Docx Generation Workflow

### Step 1: Read the docx SKILL.md first
```bash
view /mnt/skills/public/docx/SKILL.md
```

### Step 2: Determine role framing
Read the job description. Identify:
- **Primary emphasis** (engineering / data / strategy / client delivery / AI / product)
- **Key credentials to lead with** from the career facts above
- **ATS keywords** to embed in summary and skills rows

### Step 3: Write resume.js using the standard template

**Color palette (always use these):**
```javascript
const NAVY = "1F3864";
const DARK = "2E2E2E";
const GRAY = "555555";
const LIGHT_GRAY = "888888";
const ACCENT = "1A6496";
```

**Standard structure:**
1. Name (size 56, NAVY, bold, centered)
2. Tagline (size 21-22, ACCENT, bold, centered)
3. Contact line (size 18, GRAY, centered) with bottom border
4. Professional Summary
5. Core Competencies (skillRow format)
6. Professional Experience (jobHeader + bullets)
7. Education
8. Certifications

**Page setup (always US Letter):**
```javascript
page: {
  size: { width: 12240, height: 15840 },
  margin: { top: 864, right: 1080, bottom: 864, left: 1080 }
}
```

**Numbering config (always include):**
```javascript
numbering: {
  config: [{
    reference: "bullets",
    levels: [{
      level: 0, format: LevelFormat.BULLET, text: "\u2022",
      alignment: AlignmentType.LEFT,
      style: { paragraph: { indent: { left: 360, hanging: 200 } } }
    }]
  }]
}
```

### Step 4: Run and validate
```bash
node resume.js
python /mnt/skills/public/docx/scripts/office/validate.py output.docx
cp output.docx /mnt/user-data/outputs/
```

### Step 5: Present the file
Use `present_files` tool with the output path.

## Editing Existing Resumes

For targeted edits (tagline change, add a bullet, update a date):
```bash
python /mnt/skills/public/docx/scripts/office/unpack.py input.docx unpacked/
# edit XML using str_replace
python /mnt/skills/public/docx/scripts/office/pack.py unpacked/ output.docx --original input.docx
cp output.docx /mnt/user-data/outputs/
```

**Key XML patterns:**
- Text is in `<w:t>` tags
- Use `&amp;` for `&`, `&#x2019;` for `'`, `&#x2014;` for `—`
- Never use `\n` — use separate Paragraph elements
- Smart quotes: `&#x201C;` / `&#x201D;` for double quotes

- **Never over-index on a single implementation** — Aladdin Provider Model should be one example among several, not the headline
